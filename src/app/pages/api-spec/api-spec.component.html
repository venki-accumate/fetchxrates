Quickstart

Make your first API call in under 5 minutes.

1

Sign up and get your API key

Create an account at {{ appUrl.replace('https://', '') }} and choose a plan. Once subscribed, go to API Keys in the dashboard to generate your API key. It starts with fxr_ prefix.

open_in_new View pricing plans
2

Make your first request

Call the /api/rates endpoint with your API key and email to retrieve the latest exchange rates. The x-api-key and x-email headers are required on every request (except /docs and /healthcheck).

terminal cURL 
curl -X GET \
  "{{ apiUrl }}/api/rates?from=AUD&to=USD" \
  -H "x-api-key: YOUR_API_KEY" \
  -H "x-email: you@example.com"
3

Handle the response

A successful response returns 200 OK with a JSON body containing the base currency, dates, and rates.

output Response 200 OK

  "rates": {
    "USD": 0.6432
  }
}

Authentication

All FetchXRates API requests require an API key and your registered email address.

Type API Key
Header 1 x-api-key: YOUR_API_KEY
Header 2 x-email: you@example.com
Required Yes — all endpoints except /docs and /healthcheck
info

Both x-api-key and x-email headers must be sent with every authenticated request. The email must match the account that generated the API key.

API keys start with fxr_ and can be generated from the API Keys section of your FetchXRates dashboard.

How to authenticate

Include both headers in every request:

Key tips

  • API keys are permanent and do not expire — but you can regenerate them from the dashboard at any time.
  • Never expose your API key in client-side code or public repositories.
  • A 401 Unauthorized response means the API key is invalid or missing.
  • A 403 Forbidden response means your plan does not include access to the requested feature.
  • A 429 Too Many Requests response means you've exceeded your monthly request limit.
  • Each subscription plan has a request limit — check your dashboard for current usage.

Error responses

401 Missing or invalid x-api-key or x-email header
403 Feature not available on your plan (e.g. publication access, currency limit exceeded)
429 Monthly request limit reached — upgrade plan or wait for next billing cycle

Rate Limits

API usage is governed by your subscription plan. Limits are enforced per billing cycle.

Requests Monthly request cap (e.g. 2,000 / 10,000 / 20,000). Returns 429 when exceeded.
Currencies Max target currencies per request (e.g. ?to=USD,SGD,EUR). Returns 403 when exceeded.
Date Range Max date span per request (e.g. 1–5 years depending on plan). Returns 403 when exceeded.
Excel Rows Max rows per Excel conversion request (e.g. 500–20,000). Returns 403 when exceeded.
Schedules Max active automated delivery schedules (e.g. 2–10). Returns 403 when exceeded.
Publication Access to published central bank rate data. Available on Growth and Professional plans only.
